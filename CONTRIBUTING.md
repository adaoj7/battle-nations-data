# Contributing

Keep changes small, evidence-backed, and unit-focused.

## Preferred changes

- fix one unit's stats or metadata
- add one missing status effect
- add one missing attack pattern
- correct one targeting or resistance rule

## Required PR contents

- unit name and ID
- exact fields changed
- source links
- short explanation of the correction
- validation output

## Canonical vs generated

- Edit `data/units/<id>.json`.
- Do not hand-edit `data/units.json`.
